Intro:
If you’re in the developer space right now and you’re anything like me, you’re building something new every other week. But every new codex or claude code session adds to the stack of unfinished projects and unorganized context. To fix that, I’m building Topside.

What It Is:
Topside is a free, open-sourced project layer for builders and agents. It brings your projects, tasking, and documentation all into one place. Just link it to a project folder and it’s all set up. And because this is all stored in markdown, agents can fully interact with this environment. Just prompt to add to your project’s to-do or documentation.

Every part of the UI is intentionally simple. Your projects are all stored on the left sidebar. In the middle are the selected projects tasking, and can be synchronized to a real to-do markdown file in your codebase. Tasks can be grouped into headers, and are stored in the changelog section below when completed. On the right is the selected projects documentation. Add a note manually or link one directly from the codebase. 

It’s also built on Rust to stay lightweight, fast, and agent friendly.
