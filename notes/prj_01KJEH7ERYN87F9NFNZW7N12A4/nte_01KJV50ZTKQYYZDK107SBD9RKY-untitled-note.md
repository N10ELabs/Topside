Intro:
If you’re in the developer space right now and you’re anything like me, you’re building something new every other week. But every new codex or claude code session adds to the stack of unfinished projects and unorganized context. To fix that, I’m building Topside.

What It Is:
Topside is a free, open-sourced project layer for builders and agents. It brings your projects, tasking, and documentation all into one place. Just link it to a project folder and it’s all set up. And because this is all stored in markdown, agents can fully interact with this environment. Just prompt to add to your project’s to-do or documentation.

Walkthrough:
Every part of the UI is intentionally simple. Your projects are all stored on the left sidebar. In the middle are the selected projects tasking, and can be synchronized to a real to-do markdown file in your codebase. Tasks can be grouped into headers, and are stored in the changelog section below when completed with an easy way to export. On the right is the selected projects documentation. Add a note manually or link docs directly from the codebase.

It’s also built in Rust to stay lightweight, fast, and agent friendly.

Conclusion:
So that’s an early preview of Topside. It’s incredibly simple but it’s helpful for my workflow and hopefully some of you can get value from it too. One of the advantages of being a solo developer is owning the architecture end to end. You can tailor the product to your workflow and move a lot faster than large companies or even small startups. I plan on adding as much functionality as is useful to me and potential users if this gains traction.

And again, this is free so please go try it out! All I ask is to give the repo a star and like this video. Thanks for watching!
